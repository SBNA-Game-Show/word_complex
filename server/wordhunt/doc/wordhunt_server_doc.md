# Word Hunt Main Backend Components

## wordhuntcontroller

- Handling incoming HTTP requests
- Validating request parameters
- Calling appropriate services
- Returning API responses
- Managing API-level errors

### wordhuntcontroller Methods

| Method              | Description                                                                                                                      |
| ------------------- | -------------------------------------------------------------------------------------------------------------------------------- |
| `findPOSEnglish()`  | Retrieves the given storyId from req.query and calls service layer with the storyId. Returns Tokenized Passage English Version   |
| `findPOSSanskrit()` | Retrieves the given storyId from req.query and calls service layer with the storyId. Returns Tokenized Passage Sanskrit Version. |

---

## findNounsVerbsAndAdjectivesGameEnglish && findNounsVerbsAndAdjectivesGameSanskrit Service

The core business logic layer responsible for retrieving the raw data from tokenized story:

- Processing story information
- Managing story transformation workflows

### findNounsVerbsAndAdjectivesGameEnglish && findNounsVerbsAndAdjectivesGameSanskrit Methods

| Method                        | Description                                                                                                |
| ----------------------------- | ---------------------------------------------------------------------------------------------------------- |
| `findNounVerbAndAdjEnglish()` | processes the retrieved story extracts required data and sends back response to findPOSEnglish endpoint.   |
| `getStoryById()`              | given a storyId this endpoint retreives the story from tokenized_stories collection.                       |
| `extractRequiredData()`       | helper function. Given raw data this funcction extracts only the required data for english version passage |
| `breakPassageToArray()`       | helper function. Given english passage in string format breaks it into array.                              |
| `cleanTokenizedArray()`       | helper function. Given an tokenized array removes all empty POS tokens                                     |

---

## gamecontroller

Responsible for adding story information and Game Meta generated by Noun, Verb and Adjective Game:

- Handling incoming HTTP requests
- Validating request parameters
- Calling appropriate services
- Returning API responses
- Managing API-level errors

### gamecontroller Methods

| Method              | Description                                                                                                                                                        |
| ------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `addStoryInfo()`    | Given gameId, storyId, number of nouns, verbs, adjectives and hints. Initializes the Story Info class to store to repository returns a success or failure response |
| `addGameData()`     | Given Game meta initializes the game data and returns a success or failure response.                                                                               |
| `fetchPlayerData()` | Given gameId, storyId and playerName retreives the player information from regarding the particular game played.                                                   |

---

### gamecontroller Service

Responsible for enriching token data with dictionary information:

- Retrieving definitions
- Adding word relationships
- Connecting linguistic metadata with dictionary resources

#### Methods

| Method            | Description                                        |
| ----------------- | -------------------------------------------------- |
| `getDefinition()` | Retrieves dictionary definitions for a given word. |
| `getSynonyms()`   | Retrieves synonym information for English words.   |
| `getAntonyms()`   | Retrieves antonym information for English words.   |
| `lookupWord()`    | Searches dictionary sources for word information.  |

### StoryRepository

The database access layer responsible for:

- Communicating with MongoDB
- Executing database operations
- Persisting and retrieving story information

#### Methods

| Method       | Description                                      |
| ------------ | ------------------------------------------------ |
| `findById()` | Retrieves a story document using its identifier. |
| `findAll()`  | Retrieves available stories from the database.   |
| `save()`     | Stores a new story document.                     |
| `update()`   | Updates an existing story document.              |
| `delete()`   | Removes a story document from the database.      |

---

### TokenRepository

Responsible for managing tokenized story data:

- Storing NLP output
- Retrieving token metadata
- Updating processed linguistic information

#### Methods

| Method                  | Description                                         |
| ----------------------- | --------------------------------------------------- |
| `saveTokens()`          | Stores generated token information in the database. |
| `findTokensByStoryId()` | Retrieves tokens associated with a story.           |
| `updateTokenMetadata()` | Updates POS, lemma, and morphology information.     |
| `deleteTokens()`        | Removes tokenized data for a story.                 |

---

### PlayerGameRepository

Responsible for managing player game information:

- Saving game progress
- Retrieving player statistics
- Maintaining leaderboard information

#### Methods

| Method                | Description                                     |
| --------------------- | ----------------------------------------------- |
| `saveGameResult()`    | Stores completed game information for a player. |
| `getPlayerProgress()` | Retrieves player progress for a selected story. |
| `updateScore()`       | Updates player scores and achievements.         |
| `getLeaderboard()`    | Retrieves leaderboard information.              |
